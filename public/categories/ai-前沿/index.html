ReAct：让大模型边想边做（论文 2210.03629 解读）

论文：ReAct: Synergizing Reasoning and Acting in Language Models（arXiv:2210.03629） 如果你只想记一句话： ReAct = 推理（Reasoning）+ 行动（Acting）+ 观察反馈（Observation）。 为什么会有 ReAct？ 在 ReAct 之前，主流有两条路： 只强调推理（如 CoT） 优点：思路看起来更完整 问题：容易“在错误前提上越想越远” 只强调行动（直接调用工具/环境操作） 优点：执行快 问题：容易盲动，缺少全局判断 ReAct 的关键创新，就是把这两条路接成闭环： 思考 → 行动 → 观察 → 修正思考 不是一次性想完再做，而是边做边改。 ReAct 的核心机制（人话版） 在每一步里，模型会做三件事： Thought（思考）：现在我该怎么走下一步？ Action（行动）：去查资料 / 调工具 / 执行操作 Observation（观察）：拿到外部反馈，再决定是否修正 这意味着模型不再只靠“脑补”，而是持续用外部世界校准自己。 和 CoT 的本质区别 CoT 更像： 我先在脑子里一路推到答案 ReAct 更像： 我先想一步，做一步，看结果，再修正 所以 ReAct 在真实任务中通常更抗偏航，因为它有“回看”和“纠错”。 ...
