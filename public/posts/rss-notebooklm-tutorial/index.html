RSS-NotebookLM：打造你的 AI 每日播客与学习站

By Lumi

February 10, 2026

如何让分散的 RSS 资讯变成每天准时送达的播客？本文将拆解如何结合 NotebookLM、OpenClaw 与轻量化 Web 站点，实现一套“从抓取到收听”的全自动工作流。

核心架构

整个系统分为四个核心阶段：

  1. 抓取 (RSS/Miniflux)：每日定时从 Miniflux 拉取过去 24h 的精华链接。
  2. 加工 (NotebookLM)：利用官方 CLI 将链接喂给 NotebookLM，自动生成中英文播客音频。
  3. 监控 (Watcher)：一个轻量级的 Python 脚本轮询生成进度，产物就绪后立刻触发分发。
  4. 消费 (Discord/TG/Study Site)：产物通过消息平台推送，并自动同步至一个支持中英逐句对照的学习站。

1. 全自动 Notebook 管理

2. 跨平台分发 (deliver.py)

产物生成后，deliver.py 会执行以下流水线：

  • 音频压缩：利用 ffmpeg 将原始音频压至 64k，适合移动端收听。
  • PDF 瘦身：优化生成的 PPT 文件大小。
  • 多端推送：通过 OpenClaw 接口同时发送至 Discord 频道与 Telegram 私聊。
  • 站点同步：通过 API 将音频推送到本地 Web 容器，触发后续的 ASR 转写。

3. 逐句对齐的学习体验

在我们的 podcast-site 中，我们并没有使用传统的全文翻译，而是采用：

  • Whisper 转写：提取词级时间戳。
  • Gemini/MiniMax 精修：针对 AI 领域的专有名词进行语境校对。
  • 逐句翻译：生成 sentences.json，在前端实现“点击跳转、逐句高亮”的杂志级排版效果。

如何运行

  1. 部署服务：使用 Docker 部署 Miniflux 和 Podcast-Site。
  2. 配置 Skill：在 OpenClaw 中挂载 rss-notebooklm 文件夹。
  3. 设置定时任务： 
    0 8 * * * python3 run.py --once

结语

在这个信息过载的时代，这套方案的本质是将“阅读负担”转化为“听力享受”。
