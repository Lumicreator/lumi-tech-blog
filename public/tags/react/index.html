别再只会 CoT：ReAct 论文 2210.03629 的硬核拆解与落地

论文：ReAct: Synergizing Reasoning and Acting in Language Models（arXiv:2210.03629） 先把结论摆前面： ReAct 的价值不在“多一个提示词技巧”，而在把 LLM 从“静态回答”升级成“闭环决策系统”。 0. 为什么我说“只会 CoT 已经不够了” 在真实任务里（检索、工具调用、网页交互、流程决策），你会遇到三类典型失败： 推理正确，但事实错（幻觉） 计划看着很完整，但执行后不回看（错误传播） 会调用工具，但不知道何时该停、何时该改方向 CoT 主要解决第 1 层“展开推理”的问题； ReAct 解决的是第 2、3 层“推理-执行-反馈”闭环问题。 1. 论文到底提出了什么（不是口号版） 论文做的事可以压成一句工程话： 把 LLM 的输出拆成 Thought（内部推理）和 Action（可执行动作），并把外部环境返回的 Observation 喂回下一轮推理。 形成循环： Thought → Action → Observation → Thought → … → Final Answer 这比“让模型多想想”多了两件关键能力： 对外部世界取证（不只靠参数记忆） 基于反馈在线修正策略（不是一条路走到黑） 2. ReAct 为什么比 CoT 更像“可用系统” CoT 的强项 内部推理展开，适合算术/符号推理等“封闭题目” CoT 的弱项 外部事实不可验证 一旦前提错，后面会很“自洽地错下去” ReAct 的强项 推理过程中允许查证 执行后可回看并修正 轨迹可审计（这点对 Agent 运维特别重要） ...
